## Key Design Decisions

### 1. Language: Python (Kept)

The project stays in Python (pandas, matplotlib, pdfplumber). The Attempt 2 framework's R-specific configs (r-reviewer, r-code-conventions) are converted to Python equivalents.

- **PDF extraction:** Keep the processed CSVs from Attempt 1 as input (no need to re-extract). Copy the input PDFs for provenance.
- **Analysis + figures:** Python scripts (pandas, matplotlib/seaborn) in `scripts/python/`.
- **Report:** Use Quarto (.qmd) → PDF instead of markdown → ReportLab.

### 2. Output Format: Markdown Report → Quarto PDF

- Primary artifact: `Quarto/report.qmd` → renders to PDF
- Figures generated by Python scripts, saved as PNG (300 DPI) in `Figures/`
- Tables generated by Python scripts as CSV/markdown, included in Quarto
- All .claude/ configs updated for Python (not R)

### 3. What to Copy from Attempt 1 (Read-Only)

| Source | Destination | Rationale |
|--------|-------------|-----------|
| `data/input/*.pdf` (7 PDFs) | `data/input/` | Source provenance |
| `data/processed/*.csv` (5 CSVs) | `data/processed/` | Clean data; no need to re-extract |
| `literature/*.md` (3 files) | `literature/` | Research content to reuse |
| `report_infant_mortality_wisconsin.md` | `reference/` | Content reference (not the primary artifact) |
| `figures/*.png` (4 figures) | `reference/figures_attempt1/` | Visual reference for R figure matching |

### 4. What NOT to Copy

- Python scripts (will be rewritten in R)
- `vendor/` (ReportLab; not needed with Quarto)
- `MISC/` (archived drafts)
- `claude-code-my-workflow/` subfolder (CDC data — separate project)
- `.claude/` from Attempt 1 (using Attempt 2's framework)
